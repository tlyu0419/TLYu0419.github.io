Crawl Google Play APP Comments Using requests.post with Python

  • Author: tlyyu0419
  • Date: 2024-12-15 21:15:20

When we want to develop a web crawler program, we will check the request URL, headers, and payload data through DevTools. However, the Google Play website is an interesting case in which we can’t get the comment data so smoothly.

Let me show you what I’m talking about. Here is the comment data at the left side, and we can see that the comment data is from the batchexecute?rpcidsxxxxxxx URL link at the right side.

And then we can check the request url, headers and payload at the title(標題) and payload(承載) sheet in DevTools like this.

I try to copy these information and send a request but it return a 400 status code.

Strangely, if we post the data in url code format, they are equivalent, but it works! What happened?

Does it only accept the payload data in url code format? Let’s try to transform our payload data in dictionary to url code by urllib.parse.urlencode function. But it still failed.

So, let’s check waht’s the difference about the payload data. We can see that there is a small difference between them is that there’s no ‘%5C’ in Method 2. And %5C in url code queal to \. So this means when python dealing with \ python automate skip them, and this lead to the failed request.

So, we can tell Python read these string in raw string literals, so that python will not skip them, like this. And it finally works!
